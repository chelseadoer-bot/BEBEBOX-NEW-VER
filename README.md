# 베베박스 2.0 (재설계 프로토타입)

육아일기 기반 베이비 선물 레지스트리의 **UX 전면 재설계 버전**.
핵심 기능은 유지하되 디자인 시스템·고객 여정을 처음부터 다시 구성했다.
백엔드/실결제/LLM 없이 **정적 + 목업 데이터(localStorage)** 로 동작 → 어디서나 즉시 배포·검증 가능.

> 기존(1.0) 서비스와 완전히 분리된 별도 저장소. 1.0은 그대로 두고 여기서 새로 실험한다.

## 화면 3종
| 파일 | 화면 | 내용 |
|---|---|---|
| `index.html` | 사용자(부모) 앱 | 둘러보기 데모 · 온보딩 · 홈(오늘 할 일 루프) · 일기 · 위시/받은 선반 · AI 스튜디오 · 캔디 지갑 |
| `share.html` | 지인 공유 | 일기 우선 + 선물 보드(가격대) + 받은 선반 + 축하 한마디 + 2차 가입 훅 |
| `admin.html` | 운영자 | KPI · 쿠폰 지급 대기열 · 회원/세그먼트 · 전환 펀넬 |

## 로컬 실행
정적 파일이라 서버가 필요 없지만, 모듈 경로상 간단한 정적 서버 권장:
```bash
python -m http.server 8090
# http://localhost:8090/index.html  (·/share.html ·/admin.html)
```

## 배포
순수 정적 사이트. 아래 중 아무거나:
- **Netlify / Vercel / GitHub Pages**: 폴더(또는 repo) 연결만 하면 끝
- **Render**: `render.yaml`(Static Site) 포함 — repo 연결 시 자동 인식

## 파일 구조
```
index.html · share.html · admin.html
assets/
  app.css       디자인 시스템(토큰·컴포넌트)
  store.js      목업 데이터 + 유틸(공유)
  app-user.js   사용자 앱 로직
  share.js      공유 페이지 로직
  admin.js      운영자 로직
REDESIGN.md     전문가 진단 + 개발 우선순위(P0~P3)
```

## 실연동 지점(배포 시 키만 연결)
- 카카오 로그인/공유, 키디키디 상품·결제, 생성 AI(LLM) → 현재는 **데모/스텁**(화면에 `데모`·`stub` 표기).
- 결제 자동 인증/복귀는 로드맵 P1 (REDESIGN.md 참고).
